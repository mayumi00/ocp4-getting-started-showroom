== OpenShift プロジェクトについて

プロジェクトは、OpenShift における基本的な概念であり、デプロイメントの整理と管理に役立ちます。OpenShift プロジェクトを使用すると、ユーザーグループ（または個々のユーザー）は、他のグループから独立してコンテンツを整理および管理できます。各プロジェクトには、独自のリソース、ポリシー（アクションを実行できるユーザーと実行できないユーザーを定義）、制約（リソースのクォータや制限など）があります。

プロジェクトは、あなた（またはあなたのチーム）が作業に使用しているすべてのアプリケーションサービスとエンドポイントを包む「ラッパー」と考えてください。OpenShift プロジェクトは、Kubernetes の名前空間上に構築されていますが、プロジェクトの抽象化により、管理者は強化されたロールベースアクセス制御 (RBAC) などの追加機能を利用できます。

このセクションでは、Web コンソールとコマンドラインインターフェースの両方を使用してプロジェクトを操作する方法について説明します。すべての用語が理解できなくてもご安心ください。これらの概念については、ワークショップ全体を通して詳しく説明します。

== Web コンソールでプロジェクトを確認する

*{ocp4_starter_project}* というプロジェクトがすでに作成されています。まずは、link:{ocp4_starter_console_url}[Web コンソール,window='_blank']からプロジェクトを確認してください。

OpenShift Web コンソールにログインしたら、プロジェクトのドロップダウンからプロジェクトを探して選択します。プロジェクトの概要を確認すると、プロジェクトのリソースとアクティビティの概要が表示されます。

=== Topology View

プロジェクト内のアプリケーションとそのコンポーネントを視覚的に表示するTopologyパースペクティブについて理解を深めましょう。

左側のメニューから *Workloads* に移動し、*Topology* に移動します。

[.bordershadow]
image::openshift-console-topology.png[link="self",window=_blank]

まだ何もデプロイされていないため、アプリケーションやコンポーネントは表示されません。ワークショップを進めていくと、この *Topology* Viewでアプリケーションのコンポーネントを視覚化し、操作できるようになります。

== コマンドラインでのプロジェクトの操作

強力な自動化機能と高度な機能へのアクセスを提供するOpenShift CLIを使用してプロジェクトを操作することもできます。

Webターミナルで、次のコマンドを使用してプロジェクトに切り替えます。

[source,role="copypaste",subs="attributes"]
----
oc project {ocp4_starter_project}
----

次のような出力が表示されます。

[.console-output]
[source,subs="attributes"]
----
Now using project "{ocp4_starter_project}" on server "{ocp4_starter_api_url}"
----

このコマンドは、現在のコンテキストを割り当てられたプロジェクト内で動作するように設定し、特に指定しない限り、以降のすべての `oc` コマンドはこのプロジェクトスコープ内で実行されます。
