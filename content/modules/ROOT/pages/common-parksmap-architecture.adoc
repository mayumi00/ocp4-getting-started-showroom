= ParksMap のアーキテクチャ
:navtitle: ParksMap のアーキテクチャ

include::vars.adoc[]

このセクションでは、このワークショップ全体で使用する ParksMap アプリケーションのアーキテクチャについて説明します。アーキテクチャを理解することで、開発者の視点から実行するタスクをより深く理解できるようになります。ParksMap は、マイクロサービスアーキテクチャを用いて構築された、多言語対応の地理空間データ可視化アプリケーションです。異なるプログラミング言語とフレームワークを用いて開発された複数のサービスで構成されています。

[.bordershadow]
image::roadshow-app-architecture.png[align="center",link="self",window=_blank]

メインサービスは、2つの主要コンポーネントを持つウェブアプリケーションです。

1. **サーバーサイドコンポーネント**: 複数の独立したバックエンドサービスによって提供される地理空間 API を集約します。
2. **クライアントサイドコンポーネント**: JavaScript で構築され、地図上に地理空間データを視覚化し、WebSocket を使用してサーバー側と通信し、地図をリアルタイムで更新します。

以下のサービスを含む、さまざまな地図情報と地理空間情報を提供する独立したバックエンドサービスのセットをデプロイします。

* 世界中の国立公園
* 北米のメジャーリーグのスタジアム

このアプリケーションのオリジナルのソースコードは、link:{ocp4_starter_roadshow_url}[here,window=_blank] にあります。

ParksMap Web アプリケーションのサーバーサイドコンポーネントは、利用可能なすべてのバックエンドへの通信ゲートウェイとして機能します。これらのバックエンドは、OpenShift が提供するサービスディスカバリ（Service Discovery）メカニズムを使用して動的に検出されます。これについては、以降のセクションで詳しく説明します。
