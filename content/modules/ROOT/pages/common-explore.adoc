OpenShift は、プラットフォームを操作するための 2 つの主要なインターフェースを提供しています。視覚的な管理のための Web ベースのコンソールと、スクリプトや高度な操作のためのコマンドラインインターフェースです。このセクションでは、両方のツールに効果的にアクセスして使用する方法を学習します。

[#the_web_console]
== Web コンソール

OpenShift コンソールは、プロジェクト、アプリケーション、クラスターリソースを管理するための直感的なグラフィカルインターフェースを提供します。最新の Web ブラウザーからアクセスでき、一般的な管理タスクと開発タスクを簡素化するように設計されています。

=== OpenShift Web コンソールへのアクセス

[.bordershadow]
image::openshift-console-login.png[width=500,link="self",window=_blank]

OpenShift コンソールにアクセスするには、ブラウザで以下の URL にアクセスし、指定された認証情報を使用してログインしてください。

*コンソール URL:* link:{ocp4_starter_console_url}[{ocp4_starter_console_url},window='_blank']

*ユーザー名:* `{user}`

*パスワード:* `{ocp4_starter_password}`

ログインすると、メインダッシュボードが表示され、プロジェクトを確認したり、プラットフォームの各機能を操作したりできます。

[.bordershadow]
image::openshift-console-dashboard.png[link="self",window=_blank]

[#command_line_interface]
== コマンドラインインターフェース

Webコンソールは視覚的な管理に最適ですが、OpenShift CLI (`oc`) は強力な自動化機能と高度な機能へのアクセスを提供します。CLIはGo言語で記述された単一の実行ファイルで、Microsoft Windows、macOS、Linuxで利用できます。

=== CLIのインストールの確認

CLIはワークショップ環境にプリインストールされています。利用可能であること、およびバージョンを確認してください。

[source,role="copypaste",subs="attributes"]
----
oc version
----

次のような出力が表示されます。

[.console-output]
[source,subs="attributes"]
----
Client Version: {ocp4_starter_cli_version}
Kustomize Version: v5.5.0
Kubernetes Version: v1.32.6
----

=== CLI を使用したログイン

コマンドラインを使用して OpenShift クラスターに認証します。

[source,role="copypaste",subs="attributes"]
----
{ocp4_starter_login}
----

このコマンドは、ユーザー名とパスワードを使用してクラスターへの認証を行います。

NOTE: 「証明書は不明な認証局によって署名されています」というメッセージが表示された場合、このワークショップ環境では無視してかまいません。
これは、クラスターの API 証明書が自己署名されていることを示しています。これは開発環境やテスト環境では一般的です。

[.bordershadow]

TIP: 認証にはトークンを使用することもできます。トークンを取得するには、OpenShift Web コンソールで **Copy login command** オプションを使用します (図を参照)。

[.bordershadow]
image::openshift-console-login-command.png[link="self",window=_blank]

=== アクセスの確認

適切に認証され、割り当てられたリソースにアクセスできることを確認しましょう。

まず、ログイン認証情報を確認してください。
[source,role="copypaste",subs="attributes"]
----
oc whoami
----

ユーザー名が表示されます。

[.console-output]
[source,subs="attributes"]
----
{ocp4_starter_username}
----

次に、自分に割り当てられたプロジェクトが表示されていることを確認してください。
[source,role="copypaste",subs="attributes"]
----
oc projects
----

以下の例のような出力が表示されます。

[.console-output]
[source,subs="attributes"]
----
You have one project on this server: "Workspace ({ocp4_starter_project})".

Using project "{ocp4_starter_project}" on server "{ocp4_starter_api_url}".
----

完了です！これで、Webコンソールとコマンドラインインターフェースの両方からOpenShiftにアクセスできるようになりました。以降のセクションでは、これらのツールを使用して、プラットフォーム上でアプリケーションをデプロイおよび管理する方法を説明します。
